11    Reports

 

This section describes how to generate Reports in QuatroShelter by applying simple and advanced criteria. You will also learn how these criteria can be saved as Templates for future use.

 

11.1     Accessing the Reports Page

The Reports module of QuatroShelter can be accessed by login in to the QuatroShelter Web-site and following the QGReport Runner link to the Reports page.

 

 

 

 

 

 

11.1.1 Reports from Home Page

From the Home Page, you may click on the Reports icon or Reports link.

 

11.1.2 Reports from Menu Bar

The Menu Bar is visible from all pages in QuatroShelter, making Reports and other modules accessible with one click of the mouse.

 

11.1.3 Reports from Navigation Tree

The Navigation Tree is displayed in most pages of QuatroShelter. To access the Reports Page click on the “Reports” node of the tree, or click on reports that you may have already accessed.

 

 

11.2     Reports and Templates

The Reports Page shows all Reports and Templates accessible to the user.

To access a report, you may click on either the report’s name or a report template.

The system will display the Report runner screen with the various filters and criteria fields to be filled before running the report.

If the user starts a report by clicking on a template, the Report runner screen automatically populates any filters or criteria saved with the template.

Templates provide an easy way to saved report filters and criteria that can be re-used whenever the report needs to be replicated.

 

 

 

11.3     Running a Report

To run the report the user must enter any mandatory fiters. Failing to do so will generate a report error.

Mandatory filters vary depending on the type of report being generated.

 

 

The user may also add additional criteria to further filter the data being reported.

 

To generate the report the user must click on the Submit button.

 

11.4     Filters

 

11.4.1 Date Range

Applying a Date Range Filter can aid a user focus its report on a time-frame.

The date format is DD/MM/YYYY (Day, Month, Year), where the End Date must be greater than the Start Date.

The user may use the calendar button next to the date fields to chose the dates to be reported on.

 

 

11.4.2 Pay Period

Some reports may be better suited to run based on a pay period date range. The Pay Period is based on the payroll systems payroll cycle.

 

11.4.3 Org Selection

The Org Selection enables the user to filter the reported data based on the Organization Tree. The user may select a high lvel Org such as “Citizen Focused Service A” or an Org lower on the tree such as “EMS Special Ops Marine Unit”.

Please keep in mind that regardless of the filter chosen by the user, the data reported is also filtered by the user’s security access rights.

 

 

 

11.4.4 Report Options

The Report Options field enables the user to choose various options available for specific reports.

For example the Qualification Report can be sorted by Start Date, Certification Date or Expiry Date.

 

 

11.5     Additional Criteria

This section enables the user to create queries based on field names defined in the system that are relevant to the specific report.

 

 

The use may build simple queries based on one field or complex queries based on combinations of fields and relationship.

 

The parameters needed to construct the queries are table driven and can be accessed through the drop-down tables.

The basic structure of each criterion is as follows:

1.      Relation : Typically blank, it is used to build relationship between multiple criteria. Example AND, OR.

2.       

 

3.      Field Name : Provides a list of fields names that can be used to create a query. The fields are chosen based on their relevance to the report. Example: Employee Number, District.

 

 

4.      Operator: It descrives the relationship betwee the Field Name chosen and the Value field.

 

 

5.      Value: Describes the values of the field to be incorporated into the query. It can be manualy entered or chosen from a lookup table.

 

 

11.6     Saving a Template

The Save Templated function allows the user to save queries that are likely to be used in the future.

When a Template is saved, it is added to the Report Page under the report’s name and unique template name given by the user.

 

Please note that “Public” templates are available to all QuatroShelter users. “Private” templates are only available to the person that created the template.

 

 


11.7     Report Format

The output of the report is also an option to the user. By default, reports are created in Portable Document Format (pdf), because it is the most efficient way to create a read-only report that can be shared with others.

The following are the formats available:

·         Portable Document Format (PDF)

·         Crystal R$eport

·         Excel

·         MS Word

·         Text

 

To choose the report format, click on the pulldown menu next to the field.

 

11.8     Submit Report

Once all filters are entered, the user may proceed to generate the report by clicking the Submit button.

The report will be displayed on screen based on the report format chosen by the user.

 

The user then may choose to save to disk, email or print a hard copy.
